履歴タブ

履歴タブは作成された順に全てのリクエストのリストを表示します。
閲覧可能なリクエスト:
     リクエストインデックス - 各リクエストは１から番号付けされます。
     HTMLメソッド、例えば、GETまたはPOST
     リクエストされたURL
     HTTPレスポンスコード
     HTTPレスポンスコードの短い要約
     リクエスト全体にかかった時間の長さ
     リクエストに関する アラート
     Any Notes you have added to request
     Any Tags on the request
Selecting a requests will display it in the リクエストタブ 及び レスポンスタブ above.

フィルターツールバー

A filter toolbar is provided which allows you to restrict which requests are displayed.
Clicking on the Filter button displays the 履歴フィルターダイアログ.
A summary of the filter currently applied is displayed to the right of the button.

右クリック メニュー

Right clicking on a node will bring up a menu which will allow you to:

攻撃

The Attack menu has the following submenus:

動的スキャン…

This will launch the 動的スキャン dialog which allows you to initiate an 動的スキャン with the starting point set to the request you selected.

スパイダー...

This will launch the スパイダー dialog which allows you to initiate the スパイダー with the starting point set to the request you selected.

コンテキストに含める

This menu allows you to include the selected nodes and all of their subordinates in the specified context.
You also have the option to create a new context.
　 セッションコンテキスト dialog will be displayed to allow you to make any additional changes.

コンテキストから除外

This menu allows you to exclude the selected nodes and all of their subordinates from the specified context.
　 セッションコンテキスト dialog will be displayed to allow you to make any additional changes.

Flag as context

This menu has the following submenus for each of the contexts you have defined:

ログインリクエスト

This identifies the specified node as a login request.
You may only have one node identified as such in any one context.
　 セッションコンテキスト dialog will be displayed to allow you to make any additional changes.

ログアウトリクエスト

This identifies the specified node as a logout request.
You may only have one node identified as such in any one context.
　 セッションコンテキスト dialog will be displayed to allow you to make any additional changes.

以下の処理から除外する:

このメニューには、次のサブメニューがあります。

プロキシ

This will exclude the selected nodes from the proxy. They will still be proxied via ZAP but will not be shown in any of the tabs.
This can be used to ignore URLs that you know are not relevant to the system you are currently testing.
The nodes can be included again via the セッション・プロパティ dialog

スキャン

This will prevent the selected nodes from being actively scanned.
The nodes can be included again via the セッション・プロパティ dialog

スパイダー

This will prevent the selected nodes from being spidered.
The nodes can be included again via the セッション・プロパティ dialog

タグ管理...

This will bring up the Manage Tags dialog which allows you to change the タグ associated with the request.

ノート...

This will bring up the ノートダイアログを追加 which allows you to record ノート related to the request.

削除

This will remove the node and all of its children from ZAP.
However they can be added back in, to prevent this use the 'Exclude from' menus.

ブレーク...

This will bring up the ブレークポイントダイアログを追加 which allows you to set a break point on that URL.

この履歴をスキャン

This will perform an 動的スキャン against the URL related to the selected request.

Alerts for this node

If the URL selected has アラート associated with it then they will be listed under this menu.
Selecting one of the alerts will cause it to be displayed.

再送信...

This will bring up the 再送信ダイアログ which allows you to resend the request after making any changes to it that you want to.

新しいアラート...

This will bring up the アラートダイアログを追加 which allows you to manually record a new alert against this request.

サイトタブで表示

This will show the selected message in the サイトタブ.

ブラウザで表示

既定のブラウザーで選択されたノードのURLを開きます。

anti CSRF テストフォームを生成

This will open a URL which will give you a generated form for testing for CSRF issues.
It will only be enabled for POST requests, if the API is enabled and if Java supports the opening of URLs in a browser on your platform.
