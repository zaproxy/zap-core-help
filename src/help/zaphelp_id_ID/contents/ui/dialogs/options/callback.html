Pilihan Callback Address screen

Callback Mengatasi layar memungkinkan anda untuk mengkonfigurasi alamat yang digunakan untuk mendeteksi kerentanan yang memungkinkan seorang penyerang untuk memanggil remote Url.
Di versi sebelumnya ZAP API digunakan untuk tujuan ini, tetapi dari 2.6.0 seterusnya terpisah endpoint digunakan agar sistem target tidak perlu lagi akses ke API.

Local Address (e.g. 0.0.0.0)

Alamat lokal ZAP yang akan mendengarkan untuk koneksi yang baru masuk. Nilai default dari '0.0.0.0' yang berarti ZAP akan mendengarkan pada semua alamat lokal yang tersedia.

Alamat jauh

Alamat yang akan ditentukan pada serangan yang relevan. Alamat ini harus dapat diakses dari sistem target. Anda dapat menggunakan tes URL untuk memastikan kasus seperti ini terjadi.

Acak pelabuhan

Secara default ZAP akan menggunakan sebuah port yang berbeda setiap kali dijalankan. Jika anda harus menggunakan port yang sama (contohnya untuk mengizinkan akses lewat firewall) kemudian hapus centang pada opsi ini.

Menentukan pelabuhan

Jika opsi Port Random tidak dicentang, lalu port inilah yang akan didengarkan ZAP. Ini mesti berbeda dengan port lain yang digunakan ZAP, contohnya port yang digunakan sebagai koneksi proxy.

Tes URL

Ini adalah URL uji yang dapat Anda coba akses dari sistem jarak jauh. Semua akses ke URL Uji akan dicatat dalam file log ZAP pada tingkat INFO. Jika anda menggunakan UI ZAP maka akses tersebut juga akan ditampilkan di Output tab.
