Nilalaman ng Hinihimok na Data

Nilalaman ng hinihimok na data ay uri ng Estruktural na Taga-bago na nagtutukoy sa mga path ng URL na kumakatawan sa data.

Sa 'tradisyunal' na mga web application ang istraktura ng mga aplikasyon ay isang karaniwang itinukoy ng mga URL path at ang data ay nakapaloob sa mga URL parameter at POST data.
Gaya ng URL na:

  • https://www.example.com/app/aaa?ddd=eee
  • https://www.example.com/app/aaa?ddd=fff
  • https://www.example.com/app/bbb?ddd=eee
ay kumakatawan sa Tab ng mga Site bilang dalawang 'noda' sa puno:
  • Mga Site
    • https://www.example.com
      • app
        • GET:aaa(ddd)
        • GET:bbb(ddd)
Ang mga puno ng site ay napakahalaga dahil ito ay sumasalamin sa pag-unawa ng ZAP sa istraktura ng aplikasyon.
Kung ito ay hind maganadang representasyon ng istraktura, ang ZAP ay hindi magagawang epektibong umatake sa aplikasyon.

Ilang mga aplikasyon ay kinabibilangan ng mga URL path.
Halimbawa:
  • https://www.example.com/app/company1/aaa?ddd=eee
  • https://www.example.com/app/company2/aaa?ddd=fff
  • https://www.example.com/app/company3/aaa?ddd=ggg
Ang tatlong URL na ito ay kumakatawan ng parehong pahina ngunit may iba't-ibang data, ngunit bilang default, ang ZAP ay kumakatawan sa mga ito bilang tatlong magkahiwalay na noda:
  • Mga Site
    • https://www.example.com
      • app
        • company1
          • GET:aaa(ddd)
        • company2
          • GET:aaa(ddd)
        • company3
          • GET:aaa(ddd)
Ito ay isang problema dahil ang ZAP ay aatake na ngayon sa lahat ng tatlong mga pahina kapag ito lamang ay kinakailangan umatake ng isa sa kanila.
Sa kasong ito, ang pag-aatake sa parehong pahina nang 3 beses ay hindi isang malaking problema, ngunit kung mayroon kang daan-daan o libo-libong pahina tulad nito ay makabuluhang madagdagan ang oras na abutin para ma-scan ang aplikasyon.

Sa mga tuntunin ng ZAP, ang 'kompanya' na noda ay 'nilalaman ng hinihimok na data' - mga elemento ng URL path na naglalaman ng data sa halip na kumakatawan sa bahagi ng istraktura ng aplikasyon.
Maaari mong tukuyin ang mga nilalaman ng hinihimok na data sa pamamagitan ng pagdaragdag ng aplikasyon sa Konteksto at saka pagsasaayos ng mga ito sa Tab ng mga Site 'I-flag bilang Konteksto > Pangalan ng Konteksto Noda ng hinihimok na data' right-click na aytem sa menu
Kapag nagawa mo na ito, ang mga pahina ay tamang kinakatawan bilang 1 na noda:
  • Mga Site
    • https://www.example.com
      • app
        • «kompanya»
          • GET:aaa(ddd)
Ang mga character na « at » ay ginagamit upang ipahiwatig na ito ay isang 'espesyal' na noda at ang pangalan ng noda (sa kasong ito ay 'kompanya') ay maaaring mong itakda upang ipahiwatig kung ano ang kinakatawan ng noda.
