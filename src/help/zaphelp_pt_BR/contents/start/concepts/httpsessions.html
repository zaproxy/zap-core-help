HTTP Sessions

This tool keeps track of the existing HTTP Sessions on a particular Site and allows the Zaproxy user to force all requests to be on a particular session. Basicamente ela permite que se possa alternar facilmente entre as sessões de usuário em um Site e criar uma nova sessão sem "destruir" as já existentes.

É baseada no conceito de Tokens de sessão - parâmetros de mensagem HTTP (agora apenas Cookies), que permitem a um servidor HTTP conectar uma mensagem de request com quaisquer requests ou dados armazenados anteriormente. No caso do Zaproxy, tokens de sessão foram classificados em 2 categorias: tokens de sessão default/padrão e tokens de sessão do site. Os tokens de sessão padrão são aquelas que o usuário pode definir na tela de opções e que são automaticamente considerados tokens de sessão para qualquer site (por exemplo PHPSESSID, jsessionid, etc). Os tokens de sessão de sites são um conjunto de tokens para um determinado site e geralmente são criados usando-se os menus pop-up disponíveis na Aba Params.

Esta ferramenta detecta automaticamente qualquer sessão HTTP que existe na comunicação, usando os tokens de sessão definida ou os tokens de sessão padrão automaticamente detectados. The detected sessions are shown in the HTTP Sessions Tab.

The user can, using the button available on the HTTP Sessions Tab, create a new session without destroying the existing one, or can force one of the sessions as 'active'. Quando uma sessão está 'ativa', todas as solicitações de saída enviadas para o Site correspondente são modificadas, os tokens de sessão sendo configurados para corresponder à sessão ativa. Desta forma, o usuário pode facilmente forçar algumas mensagens para ser 'parte' de uma sessão e depois alternar e enviar mensagens em outra sessão.

The HTTP Sessions tool is configured using the HTTP Sessions Options screen.
