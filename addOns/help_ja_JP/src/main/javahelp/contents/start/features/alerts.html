アラート

アラートは潜在的な脆弱性を示すもので、特定のリクエストに関連付けられています。
1つのリクエストに複数のアラートが存在する場合もあります。

アラートはリスクを示すフラグを UI 上に表示します:
      高
      中
      低
      情報
      誤検知 (偽陽性)

アラートは、アクティブスキャン、パッシブスキャン、スクリプト、アドオン (拡張機能)、または「アラートの追加」ダイアログを使用して手動で作成するなど、さまざまなZAPコンポーネントによって生成されます。 「アラートの追加」ダイアログでは、アラートの詳細/情報を更新または変更することもできます。

履歴タブでは、最も高いリスクのアラートにフラグが付けられます。
すべてのアラートはアラートタブに一覧表示され、リスクごとのアラート数はフッターに表示されます。.

ZAPが生成する可能性のあるすべてのアラートは、https://www.zaproxy.org/docs/alerts/で確認できます

アラートフィールド

以下のフィールドがサポートされています。

名前

アラートの名前。例: クロスサイトスクリプティング (反射型)

URL

アラートに関連するURL。

ノード名

URLの正規化されたバージョンで、 サイトツリーでも使用されます

リスク

アラートの相対的な深刻度。

以下のいずれか:

  • 情報
  • 低
  • 中
  • 高
リスクはZAPによって生成されたアラートに対して自動的に設定されます。 リスクを変更したい場合は、各アラートに対して手動で変更するか、 アラートフィルターを作成して自動的に変更することができます。

信頼度

アラートに対する相対的な信頼度。

以下のいずれか:

  • 誤検知 (偽陽性)
  • 低
  • 中
  • 高
  • 確認済み
ZAPは「誤検知 (偽陽性)」または「確認済み」の信頼度ではアラートを生成しません。 ただし、これらのレベルは手動またはアラートフィルター経由で設定できます。

パラメーター

攻撃されたパラメーターの名前。 アラートが特定のパラメーターに関連付けられていない場合、これは空になります。

攻撃

アラートを発見するために使用されたペイロード。 パッシブアラートの場合、これは空になります。

証拠

アラートの特定に役立ったリクエストまたはレスポンスに現れる文字列。 関連する文字列がない場合 (例: セキュリティヘッダーの欠落など)、これは空になります。

CWE ID

共通脆弱性タイプ一覧 (Common Weakness Enumeration) ID

WASC ID

Webアプリケーションセキュリティコンソーシアム (Web Application Security Consortium) ID

ソース

アラートを生成したコンポーネントと、ルールのID。

アラート参照

アラートを生成したルールのID。オプションでダッシュとアラートタイプが続きます。

ZAPによって生成されるすべてのアラートは https://www.zaproxy.org/docs/alerts/ に一覧表示されており、 それらはすべてアラート参照に基づくURLを持つ静的ページがあります。

入力ベクトル

攻撃された要素を識別するために使用されるアクティブスキャンの入力ベクトル。 パッシブアラートの場合、またはアラートが特定のパラメーターに関連付けられていない場合、これは空になります。

説明

アラートの詳細な説明。 これは同じ参照を持つすべてのアラートインスタンスで同じテキストになります。

その他の情報

アラート固有の情報。生成されたアラートごとに異なる可能性があります。

解決策

根本的な問題に対する潜在的な解決策。 ZAPはソースコードにアクセスしないため、これらの解決策は一般的なものになることに注意してください。 これは同じ参照を持つすべてのアラートインスタンスで同じテキストになります。

参照

オンラインでアラートに関する詳細情報へのリンク。

アラートタグ

アラートに関連付けられたタグ。 サポートされているタグの完全な一覧は、https://www.zaproxy.org/alerttags/に記載されています。

アラートオーバーライド

ZAPによって生成されたアラートには、生成されたアラートに関する一般的な情報と固有の情報の両方が含まれています。 固有の情報は、対象のURLやパラメーターなど、発見された潜在的な問題に直接関連するものです。 一般的な情報には、説明や関連するオンラインリソースへのリンクなどが含まれます。

「アラートオーバーライド」構成ファイルを使用することで、一般的な情報を置換または追加できます。 これにより、組織で規定されたポリシー、内部ドキュメントへのリンク、使用している特定のテクノロジーに関するアドバイスなど、独自の情報をアラートに含めることができます。

アラートオーバーライド構成ファイルは、変更したい情報のみを記述したUTF-8エンコーディングのプロパティファイルです。 '#'で始まる行はコメントとして扱われ、無視されます。

フォーマットは次の通りです。 

<alertid>.<property> = [ + | - ] <your information>
サポートされているプロパティは以下の通りです。
  • name
  • description
  • solution
  • otherInfo
  • reference
アラートタグのフォーマットは次の通りです。 
<alertid>.tag.<key> = [ + | - ] [ <value> ]
注意点：
  • タグのキーに含まれるスペースは、バックスラッシュでエスケープする必要があります。
  • タグの値は省略可能です。
例： 
# 40012 = 反射型XSS
40012.solution = http://internet.example.com/xss.html にある当社のガイドラインに従ってください。
40012.tag.Bug\ Tracker\ URL = https://example.com/issues/1234
値が '+' で始まる場合、既存の情報に追記されます。
値が '-' で始まる場合、既存の情報の前に挿入されます。
値が '+' または '-' で始まらない場合、既存の情報を置き換えます。

アラートオーバーライド設定ファイルは、 API, オプション - アラート画面 または以下の コマンドライン オプションで指定できます。 

-config alert.overridesFilename=<filename>
