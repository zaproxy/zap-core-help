スクリプト

ZAPでは、ZAPに埋め込むことができ、内部のデータ構造やクラスにアクセス可能なスクリプトをサポートしています。 これらのスクリプトにより、ZAP内からZAPを動的に拡張することができます。

ZAPは、JSR 223をサポートする任意のスクリプト言語をサポートしています。以下が含まれます：

  • ECMAScript / JavaScript (GraalVM JavaScriptアドオン経由)
  • Zest
  • Groovy https://groovy-lang.org/
  • Kotlin https://kotlinlang.org/
  • Python https://www.jython.org
  • Ruby - https://jruby.org/
  • その他多数…
⚠️ 警告 - スクリプトはZAPと同じ権限で実行されますので、信頼できないスクリプトは実行しないでください！

スクリプトタイプ

さまざまな種類のスクリプトがサポートされています：
  • アクティブルール - これらはアクティブスキャナーの一部として実行され、個別に有効化できます
  • 認証 - コンテキストに対して認証が実行される際に呼び出されるスクリプト 使用するには、コンテキストのスクリプトベース認証方式を設定する際に選択する必要があります。
  • エクステンダー - 新しい機能 (グラフィカル要素や新しいAPIエンドポイントを含む) を追加できるスクリプト
  • HTTP Sender - ZAPが送信するすべてのリクエストと受信するすべてのレスポンスに対して実行されるスクリプト。 これには、プロキシされたメッセージ、アクティブスキャン中に送信されるメッセージ、ファザーなどが含まれます
  • 入力ベクトル - ZAPが正確に何を攻撃すべきかを定義するスクリプト。 また、サイトツリーでのURL表現方法を変更することもできます
  • パッシブルール - これらはパッシブスキャナーの一部として実行され、個別に有効化できます
  • プロキシ - これらは「インライン」で実行され、すべてのリクエストとレスポンスを変更でき、個別に有効化できます。 ブレークポイントをトリガーすることもできます
  • セッション管理 - アプリケーションがセッションをどのように管理するかを定義するスクリプト
  • スタンドアロン - 自己完結型のスクリプトで、手動で開始した場合にのみ実行されます
  • ターゲット - ターゲットURLとともに呼び出され、手動で開始した場合にのみ実行されるスクリプト
注: アドオンは追加のスクリプトタイプを追加できます。これらについては、対応するアドオンのヘルプで説明されています。

ZAPスクリプトの実行方法の詳細については、スクリプトコンソールアドオンのヘルプページを参照してください。
