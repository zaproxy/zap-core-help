オプション アクティブスキャン画面

この画面ではアクティブスキャンのオプションを設定できます：

同時にスキャンするホスト数

同時にスキャンされるホストの最大数です。
この値を増やすと、ZAPが動作しているコンピュータに追加の負荷をかける可能性があります。

ホストあたりの並列スキャンスレッド数

スキャナーがホストごとに使用するスレッド数です。
スレッド数を増やすとスキャンが高速化されますが、ZAPが動作しているコンピュータとターゲットホストに追加の負荷をかける可能性があります。

最大表示件数

アクティブスキャンタブに表示される結果数です。
大量の結果を表示すると、スキャンにかかる時間が大幅に増加する可能性があります。

ルールの最大実行時間 (分、0 = 無制限)

個々のルールが実行できる最大時間 (分単位) です。ゼロは制限なしを意味します。これは、過度に時間を要しているルールを防ぐために使用できます。

ルールあたりの最大アラート数

アクティブスキャン中に任意のルールが発生させることができるアラートの最大数です。この値に達したスキャンルールはスキップされます。
注意:スレッド処理により最大値を超える可能性があります。

ルールの最大実行時間 (分、0 = 無制限)

スキャン全体が実行できる最大時間 (分単位) です。ゼロは制限なしを意味します。これは、設定された時間頃にスキャンが完了することを保証するために使用できます。

スキャン時の遅延（ミリ秒）

各リクエスト間の遅延時間 (ミリ秒単位) です。
これをゼロ以外の値に設定すると、アクティブスキャンにかかる時間は増加しますが、ターゲットホストへの負荷は軽減されます。
注意: このオプションは廃止予定であり、将来のリリースで削除されます。 代わりに、[ネットワーク] > [レート制限]オプションを使用してください。 後者のオプションはリクエスト送信レートを強制できますが、「スキャン時の遅延」では強制できません。

すべてのアクティブスキャンリクエストのヘッダーにプラグインIDを挿入する

このオプションを選択すると、アクティブスキャナーは、HTTPリクエストを送信しているスキャンルールのIDを含むリクエストヘッダー X-ZAP-Scan-ID を注入します。

Anti-CSRFトークンを処理する

このオプションを選択すると、アクティブスキャナーは必要に応じてAnti-CSRFトークンの自動リクエストを試行します。
以前は、これによりスキャナーが1つのスレッドのみを使用することが強制されていましたが、現在はそうではありません。 複数のスレッドが使用されている場合、Anti-CSRFトークンが正しく生成されているかどうかを確認することを強く推奨します。 例えば、カスタムタグを使用してレスポンス内の成功/失敗パターンを確認してください。

攻撃モードでスコープが変更された際にノードの再スキャンを促す

このオプションを選択すると、攻撃モードを選択した際に、スコープ内のノードを再スキャンするかどうかを選択するプロンプトが表示されます。
このオプションが選択されていない場合、以下のオプションがノードを再スキャンするかどうかを制御します。

攻撃モードでスコープが変更された際に常にノードを再スキャンする

このオプションを選択すると、アタックモードで動作している際に、スコープが変更された場合、スコープ内のすべてのノードが再スキャンされます。
大規模サイトでは長時間を要する可能性があるため、これは推奨されません。

既定のアクティブスキャンポリシー

アクティブスキャンを開始する際にデフォルトで使用されるスキャンポリシーです。

攻撃モードスキャンポリシー

攻撃モードでのスキャンに使用されるスキャンポリシーです。.

最大進捗チャート時間（分）

スキャン進行状況ダイアログでレスポンスコードをチャート表示する最大時間 (分単位) です。
チャートを無効にするには、このオプションを0分に設定してください。
