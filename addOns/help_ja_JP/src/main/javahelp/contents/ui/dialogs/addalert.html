アラートの追加ダイアログ

このダイアログでは、特定のリクエストに関連するアラートを手動で追加または変更できます。

フィールド

このダイアログには以下のフィールドがあります：

タイプ

アラートの種類は、事前定義された問題の種類のセットから選択できるプルダウンフィールドです。
独自のテキストを入力したり、選択した項目のテキストを変更したりすることも可能です。
既存の種類を選択すると、選択した項目に関連するテキストが「詳細」、「解決策」、「その他の情報」フィールドに自動入力されます。

リスク

リスクの深刻度を指定できるプルダウンフィールドです：
    情報
    低
    中
    高

信頼度

発見した脆弱性の有効性に対する信頼度を指定できるプルダウンフィールドです：
    誤検知 (偽陽性)後に悪用不可能と判明した潜在的な問題用
    低未確認の問題用
    中ある程度確信を持てる問題用
    高高い確信を持てる発見用
    確認済み確認済みの問題用

パラメーター

問題が関連付けられているパラメーターを指定できるプルダウンフィールドです。
このフィールドには検出されたURLおよびFORMパラメーターが事前入力されますが、独自のパラメーター名を入力することも可能です。
配列パラメーター (URLクエリコンポーネントおよび x-www-form-urlencoded リクエストボディ内) はインデックスで識別されます。例えば、 choices[]=ChoiceA&choices[]=ChoiceB を含むリクエストの場合、最初のパラメーターは choices[0] 、2番目は choices[1] として識別されます.

攻撃

パラメーターに供給されたペイロードです。これは、ペイロード/攻撃を送信するアクティブスキャンルールによってのみ設定されることが想定されています。

証拠

問題の存在を示すメッセージリクエストまたはレスポンス (ヘッダまたはボディ) の利用可能な情報です。

CWE ID

フラグが立てられた脆弱性に関連する共通脆弱性タイプ一覧 (CWE) IDです。このフィールドは整数のみ入力可能です。

WASC ID

フラグが立てられた脆弱性に関連するWebアプリケーションセキュリティコンソーシアム (WASC) IDです。このフィールドは整数のみ入力可能です。

説明

発見された問題の種類の一般的な説明です。
事前定義された種類を選択すると自動入力されますが、必要に応じて変更することも可能です。
注意: 別の種類を選択すると、変更内容は失われます。

その他の情報

発見した特定の問題に固有の情報です。
このフィールドは事前入力されません。

対策

問題の修正方法に関する推奨事項です。
事前定義された種類を選択すると自動入力されますが、必要に応じて変更することも可能です。
注意: 別の種類を選択すると、変更内容は失われます。

参考文献

選択したアラートの種類に関するインターネット上の詳細情報を指すURLです。
事前定義された種類を選択すると自動入力されますが、必要に応じて変更することも可能です。
注意: 別の種類を選択すると、変更内容は失われます。

アラートタグ

アラートには任意のキーまたは キー=値 のペアでタグ付けできます。
各アラートに任意の数のタグを追加できます。ただし、
  • 同じキーを持つ2つのタグは設定できません。
  • 各タグのキーは1024文字、値は4000文字に制限されています。
