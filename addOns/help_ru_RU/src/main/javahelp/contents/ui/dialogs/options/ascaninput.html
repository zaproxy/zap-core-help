Окно «Параметры активного сканирования: входные векторы»

Этот экран позволяет настроить активное сканирование входные векторы.
Это элементы, которые будет атаковать активный сканер.
Сканирование всех поддерживаемых элементов займет больше времени, но отсутствие сканирования некоторых элементов может привести к тому, что некоторые уязвимости будут упущены.

Строка запроса URL и узлы, управляемые данными

Пары ключ-значение в запросе URL запроса, т.е. после ?.
Если Узлы, управляемые данными определены в контексте, они будут проверены.
Добавить параметр запроса URL?
При выборе ZAP добавит параметр запроса к запросам GET, в которых изначально его не было. Это может увеличить время сканирования, однако также может выявить проблемы, которые в противном случае остались бы незамеченными.

POST Данные

Пары ключ-значение в запросе данных POST.

Элементы пути в URL-адресе запроса, т. е. элементы, разделенные /.

Запрос HTTP-заголовков.
Все запросы
Позволяет сканировать заголовки HTTP всех запросов. Не только запросы, которые отправляют параметры через запрос или тело запроса.

Включить входные векторы сценария

Если выбран этот параметр, активный сканер будет использовать любые разрешенные входные векторы скрипта.
Входные векторы скриптов — это скрипты, которые вы написали или импортировали в ZAP и которые позволяют нацеливать элементы. которые не поддерживаются по умолчанию.

Этот экран также позволяет настроить параметры, которые будут игнорироваться активным сканером.
Ссылаться на Параметр раздел в Добавить оповещение диалог для более подробной информации о названия параметров.
