Aktif Tarayıcı Ekran Ayarları

Bu ekran şunu düzeltmenize izin verir aktif tarama seçenekler:

Number of Hosts Scanned Concurrently

Aynı anda taranacak ana bilgisayar sayısının maksimum sayısı. Bunu arttırmak ZAP'ın çalıştığı bilgisayara ek yük yükleyebilir.

Concurrent Scanning Threads per Host

Tarayıcının ana bilgisayar başına kullanacağı iş parçacığı sayısı.
Konu sayısının artırılması taramayı hızlandıracaktır, ancak ZAP'ın çalıştığı bilgisayarda ve hedef bilgisayar üzerinde ilave yük bindirebilir.

Max Results to List

Aktif tarama sekmesinde gösterilecek sonuçların sayısı.
Fazla sayıda sonuç görüntüleme, bir taramanın süresini önemli ölçüde artırabilir.

Maximum Rule Duration (minutes; 0 is unlimited)

Herhangi bir kuralın birkaç dakika içinde başlayabileceği maksimum süre. Sıfır limitsiz demektir. Bu, aşırı zaman alan kuralları önlemek için kullanılabilir.

Maximum Scan Duration (minutes; 0 is unlimited)

Taramanın maksimum tarama süresi dakika olarak. Sıfır limit yok demektir. Bu özellik taramanın belirlenen sürede tamamlanması için kullanılır.

Delay When Scanning (In Milliseconds)

Milisaniye olarak her istek arasındaki gecikme.
Bunu sıfırdan farklı bir değere ayarlamak, etkin bir tarama süresini artıracaktır, ancak hedef ana bilgisayara daha az yük bindirecektir.

Inject plugin ID in header for all active scan requests.

Bu seçimle aktif tarama istenilen başlığı ekler X-ZAP-Tarama_ID hTTP isteklerini gönderen tarayıcı kimliğiyle birlikte.

Handle anti-CSRF tokens.

Bu seçenek seçilirse, etkin tarayıcı otomatik olarak istekte bulunmaya çalışacaktır anti CSRF gerektiği zaman belirtir.
Previously this would have forced the scanner to only use one thread, but that is no longer the case. You are strongly recommended to check that the anti CSRF tokens are being correctly generated if more than one thread is being used, for example using custom tags to check for success / failure patterns in the response.

In Attack Mode prompt to rescan nodes when scope changed.

If this option is selected then when you select Attack mod you will be prompted to choose whether to rescan nodes in scope.
If the option is not selected then the following option will control whether the nodes are rescanned.

In Attack Mode always rescan nodes when scope changed.

If this option is selected then when running in Attack mode all nodes in scope will be rescanned if the scope changes.
This is not recommended for large sites as it could take a long time.

Default Active Scan Policy

The Tarama Yönetmeliği that is used by default when you start an active scan.

Attack Mode Scan Policy

The Scan Policy that is used for scanning in Attack mode.

Max Progress Chart in Mins

The maximum time in minutes for which response codes will be charted in the Scan Progress dialog.
To disable the chart the option should be set to zero minutes.
