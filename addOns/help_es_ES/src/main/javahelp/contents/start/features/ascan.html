Escaneo Activo

El escaneo activo intenta encontrar vulnerabilidades potenciales usando ataques conocidos contra objetivos seleccionados.

El escaneo activo es una ataque a esos objetivos.
No debería ser utilizado en aquellas aplicaciones web que no son de su propiedad o no tiene permiso explícito.

Para facilitar la identificación del tráfico ZAP y poder configurar las excepciones en las reglas de Firewall para aplicaciones web, ZAP incluye un script "AddZapHeader.js" que se puede usar para añadir un encabezado a todo el tráfico que pasa o se origina en ZAP. eg: X-ZAP-Initiator: 3

Señalar que el análisis activo solo puede encontrar ciertos tipos de vulnerabilidades.
Las vulnerabilidades lógicas, como un control de acceso roto, no se detectaran por ningún análisis de vulnerabilidad activo o automatizado.
Siempre se deben realizar pruebas de penetración manual además de las activas. Escanea para encontrar todo tipo de vulnerabilidades.

El escaneo activo se configura usando la Pantalla Opciones Escaneo Activo.
Las reglas que se ejecutan se configuran a través de Políticas de análisis; puedes usar tantas como quieras.
